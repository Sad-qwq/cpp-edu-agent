# C++ 出题规范文档模板

> 用途：本模板用于 AI 出题知识库，可作为班级资料或公共知识库中的“出题规范”文档上传。
> 使用方式：将其中的占位内容替换为本课程实际教学要求，再上传到知识库。

## 1. 文档定位

- 课程名称：C++ 程序设计
- 适用对象：[例如：大一新生 / 中职二年级 / 非计算机专业基础课]
- 课程阶段：[例如：基础语法阶段 / 数据结构前导阶段 / 面向对象入门阶段]
- 出题目标：生成适合课堂练习、章节测验、课后作业的题目
- 语言风格：中文题干，术语准确，表述简洁，不使用与课程阶段不匹配的高级表达

## 2. 总体出题原则

1. 所有题目必须紧扣给定主题与知识点，不得跨章节随意出题。
2. 题目优先考查课堂核心目标，不考偏门技巧，不考未教学内容。
3. 题干必须清晰、可判定、可作答，避免含糊提问。
4. 选择题用于概念辨析，简答题用于原理说明，编程题用于实现与综合应用。
5. 如果知识库中出现无关代码片段、项目业务逻辑或框架代码，应忽略，不得直接改写成题目。
6. 优先使用教学讲义、知识点说明、样题库、术语说明等资料作为出题依据。
7. 对于编程题，必须保证输入、输出、题意、测试用例、自带参考解彼此一致。

## 4. 难度分层规范

### 4.1 easy

- 仅考查 1 个核心知识点
- 题干直接，不设置复杂背景
- 不要求多步推理
- 编程题只涉及基础输入输出与简单逻辑

示例：
- 判断某段 for 循环执行次数
- 写出数组遍历的基本代码
- 解释函数返回值的作用

### 4.2 medium

- 可组合 2 个相关知识点
- 允许出现简单场景化描述
- 需要学生理解概念并正确应用
- 编程题可要求简单统计、查找、条件判断

示例：
- 用函数处理数组中的最大值问题
- 比较 for 与 while 在数组遍历中的差异
- 说明二维数组下标的含义并完成简单实现

### 4.3 hard

- 允许 2 到 3 个同章节内知识点综合应用
- 强调边界情况、代码组织或解题步骤完整性
- 不能脱离当前章节突然引入未学内容
- 编程题应具备明确输入输出、边界测试和参考解

示例：
- 用函数封装二维数组处理逻辑
- 在数组统计题中结合条件判断和循环控制
- 设计一个综合练习题，要求模块化实现

## 5. 题型规范

### 5.1 选择题规范

- 推荐 4 个选项
- 只有 1 个最佳答案
- 干扰项要有迷惑性，但不能胡编乱造
- 正确答案必须与某个选项文本完全一致
- 不要用“以上都对/都不对”作为默认偷懒选项

选择题适合考查：
- 概念辨析
- 语法判断
- 常见错误识别
- 基础代码阅读

选择题不适合：
- 过度开放的问题
- 需要大量计算或长篇分析的问题

### 5.2 简答题规范

- 题干应明确要求“说明、比较、简述、分析”哪一项
- 要求聚焦，不要让学生无边界发挥
- 参考答案应覆盖关键点，而不是一句空话
- 适合考查概念理解、步骤总结、优缺点比较

简答题适合考查：
- 函数与数组的关系
- 循环结构适用场景
- 面向对象基础概念

### 5.3 编程题规范

- 必须给出明确题意
- 必须给出输入输出说明或能从题干中唯一推断
- 必须提供 code_template
- 必须至少提供 2 个 test_cases
- 必须提供 reference_solution
- 不能只给任务不给判定依据

编程题适合考查：
- 基础输入输出
- 循环与条件结合
- 数组统计、查找、遍历
- 用函数封装简单逻辑

编程题不适合：
- 超纲算法
- 依赖图形界面或外部库
- 题意模糊且无法自动判题的问题

## 6. 知识点与题目映射要求

出题时必须保证：
- 题目主题与目标知识点一致
- target_knowledge_points 只填写直接相关知识点
- 如果资料与知识点冲突，应优先遵守知识点要求
- 不得因为资料中偶然出现某段代码，就把题目偏到其他章节

错误示例：
- 主题是“数组的使用”，却围绕继承、多态、汽车类设计出题
- 主题是“函数定义”，却考 STL 容器复杂操作
- 主题是“循环”，却加入未讲授的递归优化技巧

## 7. 质量检查清单

每道题生成前后必须自检以下项目：

- 是否紧扣主题
- 是否紧扣目标知识点
- 是否符合指定难度
- 是否符合指定题型
- 是否适合课堂教学
- 是否不存在明显跑题
- 是否不存在超纲内容
- 编程题是否包含模板、测试用例和参考解
- 选择题答案是否能在选项中找到
- 简答题参考答案是否覆盖关键点

## 8. 高质量样题模板

### 8.1 选择题样例模板

- 主题：[例如：数组]
- 知识点：[例如：一维数组遍历]
- 难度：easy
- 题目：以下哪段代码能够正确遍历长度为 n 的整型数组 arr？
- 选项：
  - A. for (int i = 0; i < n; ++i) cout << arr[i];
  - B. for (int i = 1; i <= n; ++i) cout << arr[i];
  - C. while (arr[i]) i++;
  - D. cout << arr[n];
- 正确答案：for (int i = 0; i < n; ++i) cout << arr[i];
- 说明：考查数组下标范围与基础遍历写法

### 8.2 简答题样例模板

- 主题：[例如：函数]
- 知识点：[例如：函数参数传递]
- 难度：medium
- 题目：请简述数组作为函数参数传递时需要注意的问题。
- 参考答案：应说明数组传参时会退化为指针、函数内部无法直接获得原数组长度、通常需要额外传入长度参数。
- 说明：考查函数与数组结合的核心概念

### 8.3 编程题样例模板

- 主题：[例如：循环与数组]
- 知识点：[例如：数组遍历、条件统计]
- 难度：hard
- 题目：输入 n 和 n 个整数，输出其中大于平均值的元素个数。
- 代码模板：
```cpp
#include <iostream>
#include <vector>
using namespace std;

int main() {
    int n;
    cin >> n;
    vector<int> nums(n);
    // TODO: 补全代码
    return 0;
}
```
- 测试用例：
  - 输入：5\n1 2 3 4 5\n 输出：2\n
  - 输入：4\n2 2 2 2\n 输出：0\n
- 参考解：根据课程实际要求填写
- 说明：考查循环、数组读取、平均值统计与条件判断

## 9. 反例库模板

以下类型题目视为低质量题目，应避免生成：

1. 跑题题目
- 主题要求“数组”，题目却考“继承、多态、类设计”

2. 超纲题目
- 当前课程只讲基础语法，却要求使用高级 STL 算法

3. 含糊题目
- 题干没有说明输入输出，也没有明确要求结果形式

4. 不可判定题目
- 编程题没有测试用例，没有标准答案

5. 伪简答题
- 题目过于开放，任何回答都算对
